# QueueMate

## Проблема

Студенти часто витрачають час на хаотичне узгодження консультацій з викладачем або ментором через повідомлення. Через це виникають черги, дублювання записів та непорозуміння щодо часу зустрічі.

## Рішення

QueueMate — це простий REST API застосунок для бронювання консультацій. Студент може створити заявку на консультацію, переглянути список активних бронювань та скасувати запис.

## Цільовий користувач

- Студенти
- Викладачі
- Ментори
- Куратори

# User Stories

### User Story 1

Як студент, я хочу створити заявку на консультацію, щоб зафіксувати тему та бажаний час зустрічі.

### User Story 2

Як викладач, я хочу переглядати список активних заявок, щоб бачити чергу консультацій.

### User Story 3

Як студент, я хочу скасувати заявку, якщо консультація більше не потрібна.

# Нефункціональні вимоги

## Performance

Час відповіді GET-запиту на отримання списку бронювань не повинен перевищувати 300 мс при типовому навчальному навантаженні.

## Availability

Застосунок повинен бути доступним не менше ніж 99% часу протягом навчального семестру.

## Security

Усі запити повинні містити API-ключ у заголовку:

X-API-Key: secret-key

Якщо ключ відсутній або неправильний, сервер повертає статус:

401 Unauthorized

# API Contracts

## Endpoint 1: Створити бронювання

### HTTP Method

POST

### URL

/api/v1/bookings

### Request

```json
{
  "studentName": "Andrii",
  "topic": "Питання по Spring Boot",
  "requestedTime": "2026-06-10T15:30:00"
}
```

### Response 201 Created

```json
{
  "id": "b3f7a1e0-7d2c-4c44-9f3e-8c6c0c5d1111",
  "studentName": "Andrii",
  "topic": "Питання по Spring Boot",
  "requestedTime": "2026-06-10T15:30:00",
  "status": "ACTIVE",
  "createdAt": "2026-06-01T12:00:00"
}
```

---

## Endpoint 2: Отримати список бронювань

### HTTP Method

GET

### URL

/api/v1/bookings?status=ACTIVE

### Response 200 OK

```json
{
  "bookings": [
    {
      "id": "b3f7a1e0-7d2c-4c44-9f3e-8c6c0c5d1111",
      "studentName": "Andrii",
      "topic": "Питання по Spring Boot",
      "requestedTime": "2026-06-10T15:30:00",
      "status": "ACTIVE"
    }
  ],
  "totalCount": 1
}
```

# Тест-кейси

## Тест-кейс 1: Успішне створення бронювання

### Given

Користувач має правильний API-ключ.

### When

Користувач надсилає POST-запит із коректними даними.

### Then

Сервер повертає 201 Created та створює бронювання зі статусом ACTIVE.

---

## Тест-кейс 2: Час консультації в минулому

### Given

Користувач має правильний API-ключ.

### When

Користувач надсилає POST-запит із датою в минулому.

### Then

Сервер повертає 400 Bad Request.

---

## Тест-кейс 3: Запит без авторизації

### Given

Користувач не передав API-ключ.

### When

Користувач надсилає GET-запит на /api/v1/bookings.

### Then

Сервер повертає 401 Unauthorized.
